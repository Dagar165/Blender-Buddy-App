# Как подключить новую картинку призрака

Node и Python на рабочей машине нет, поэтому всё делается встроенными
средствами Windows плюс браузер для сжатия. Порядок такой.

## 1. Вырезать фон

```
powershell -File tools\cutout-mascot.ps1 -In "путь\к\5.png" -Out "5-cut.png"
```

Скрипт сам находит фон заливкой от краёв. Фоном считается только
нейтрально-серый пиксель, поэтому синее тело призрака не страдает.
Мелкие блики и «звёздочки» рядом с персонажем выкидываются автоматически.

Если сквозь прозрачную деталь (стекло очков) просвечивает шахматка,
добавь ключ `-KillInside`.

## 1а. ВАЖНО: у всех стадий должен быть ОДИН кадр

`cutout-mascot.ps1` обрезает картинку по границам персонажа и растягивает
его на 80% холста. Для одиночной картинки это правильно, а для НАБОРА
стадий — ломает всё: у кого силуэт короче, того растянут сильнее, и призрак
запрыгает при эволюции. Именно так разъехались первые пять стадий.

Поэтому набор режется одним общим кадром. Мерка снята с малыша
(`ghost-stage-1.webp`), под неё подогнаны остальные:

- высота тела — **77,6–78,0%** высоты холста
- нижний край тела — **90%**
- центр — **50%**

Проверять после нарезки, до сжатия. Разброс больше половины процента
уже заметен глазом на эволюции.

## 2. Сжать в WebP

Картинка после вырезки весит пару мегабайт — в приложение столько класть
нельзя. На машине нет ни Node, ни Python, ни ImageMagick, а System.Drawing
не умеет писать WebP. Умеет браузер, поэтому:

```
powershell -File tools\webp-server.ps1 -InDir <папка с png> -OutDir <куда>
```

затем открыть `http://localhost:8791/` и дождаться «ALL DONE» — страничка
сама переберёт все файлы, нарисует каждый на canvas 768×768, отдаст
`canvas.toBlob(..., 'image/webp', 0.9)` и отправит результат обратно.
Сервер выключится сам. Выходит 26–58 КБ при том же виде на экране.

Если запускаешь это из агента: сервер надо стартовать ОТДЕЛЬНЫМ фоновым
процессом (`Start-Job` между вызовами инструментов не выживает).

768 пикселей взяты с запасом: на главном экране призрак 240 CSS-пикселей,
на телефонах с плотным экраном это до 720 реальных.

## 3. Положить в приложение

Файл кладём в `client/src/assets/mascot/ghost-stage-N.webp`, добавляем
`import` наверху `client/src/lib/pet-config.ts` и ставим его в поле `image`
нужной стадии.

## Что сделано 21.07.2026

Стадии 2–5 перерисованы одним листом (пять фигур в ряд) и нарезаны с общим
кадром. Силуэт у всех одинаковый — растёт только внутреннее наполнение:
ровный цвет → перелив → искры → стекло со звёздами → галактика. Решение
владельца, обсуждено и закрыто. Стадия 1 (малыш в подгузнике) осталась
из первой генерации.

Исходный лист приходил размером 4096×2304 с НАРИСОВАННОЙ шахматкой,
фигура — 855 точек в высоту. Это БОЛЬШЕ, чем нужно приложению (тело
в готовом файле ~600 точек), поэтому апскейл не потребовался — только
уменьшение, а оно качество не портит.

## Одежда — отдельный порядок (21.07.2026)

Вещи пришли листом на ЧЁРНОМ фоне, поэтому у них свой резак:

```
powershell -File tools\cutout-items.ps1 -Path "лист.jpg" -OutDir <куда>
```

Он режет фон по яркости (фон 0,0,0, самое тёмное место вещи — 31),
разбирает лист на отдельные объекты и сохраняет каждый своим файлом.
`cutout-sheet.ps1` тут не годится: тот рассчитан на СВЕТЛЫЙ фон.

Дальше — примерочная:

```
powershell -File tools\fit-clothing.ps1 -Ghost <stage-2.png> -SpriteDir <вырезанное> `
  -Out preview.png -Frame 1.18 -SaveDir <спрайты> -Items "file=item-02.png;w=56;cx=50;cy=8;as=hat"
```

`-Frame 1.18` обязан совпадать с `CLOTHING_FRAME` в `shop-config.ts`:
холст одежды выше кадра призрака и прижат к его низу, потому что над
макушкой в кадре всего 12% высоты, а колпаку нужно вдвое больше.

Потом тот же `webp-server.ps1` (для одежды `-Size 900`) и `overlay:`
в `shop-config.ts`.

### Главный урок про исходники одежды

**Вещь, нарисованная отдельно «как в магазине», на голову не сядет.**
У наушников с того листа чашки развёрнуты к зрителю и нарисованы крупно:
ужми их до размера уха — и дужка станет уже головы. Это геометрия,
подгонкой не лечится. Шляпа и очки сели только потому, что нарисованы
почти анфас.

Поэтому вещь просим НА ПРИЗРАКЕ, а вторым шагом — её же отдельно, в том же
месте кадра. Промпты лежат в `JKids_Bot_промпты_картинок.md`.

## Чего ждать от исходников

Картинки из генератора приходят с нарисованной «шахматкой» вместо
настоящей прозрачности — у всех пяти присланных так. Это нормально,
скрипт с этим справляется. А вот два дефекта скрипт вылечить не может,
их надо избегать при генерации:

- **запечённое свечение вокруг персонажа** (как на 5-й стадии): оно
  смешано с фоном, вырезать начисто нельзя — остаётся жёлтое пятно
  с квадратиками. Просить картинку без свечения;
- **фон в цвет персонажа**: заливка от краёв остановится не там.
  Лучший фон — обычная серая шахматка или ровный серый.
